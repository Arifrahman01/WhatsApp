REST API Reference

Integrate WhatsApp capabilities into your application using our HTTP API. All endpoints are prefixed with /api/v1.

Authentication

Authenticate your API requests by including your API key in the request headers.

Header x-api-key: notifwa-secret-key-123
POST

/api/v1/send

Send a text message to a specific WhatsApp number. The number must be registered on WhatsApp.

Request Body (JSON)
{
  "number": "08123456789",
  "message": "Hello from NotifWA API!"
}
cURL Example
curl -X POST http://localhost:3000/api/v1/send \
  -H "x-api-key: notifwa-secret-key-123" \
  -H "Content-Type: application/json" \
  -d '{"number":"08123456789","message":"Hi!"}'
Responses
200 OK 
{
  "status": true,
  "message": "Message sent successfully"
}
GET

/api/v1/status

Retrieve the current connection status of the WhatsApp session. Returns a QR code string if disconnected.

cURL Example
curl http://localhost:3000/api/v1/status \
  -H "x-api-key: notifwa-secret-key-123"
Responses
200 OK 
{
  "status": "Connected",
  "qr": null
}
POST

/api/v1/logout

Terminate the current WhatsApp session and delete authentication data.

curl -X POST http://localhost:3000/api/v1/logout \
  -H "x-api-key: notifwa-secret-key-123"
